RFID Access Control System

IoT 디바이스와 AWS 클라우드를 연동하는 실시간 출입 제어 시스템 개발 가이드

1. 시스템 개요

이 프로젝트는 ESP8266 NodeMCU v2 마이크로컨트롤러와 PN532 RFID 카드 리더기를 클라이언트 장치로 사용하고, AWS 가상 서버(Apache, PHP, MariaDB)를 백엔드로 삼아 구축한 실시간 출입통제 시스템입니다.

사용자가 장치에 RFID 카드를 대면 클라이언트는 카드의 고유 식별코드(UID)를 읽어 서버로 즉시 전송합니다. 서버는 데이터베이스를 조회하여 정식 등록된 사용자인지(승인), 등록되지 않은 침입 시도인지(거부) 판별하여 기록을 누적하고, 결과를 아두이노 장치 및 실시간 대시보드 웹 애플리케이션으로 전송합니다.

2. 하드웨어 구성 및 핀 맵

ESP8266 NodeMCU v2 개발보드와 센서 모듈 간의 연결 배선 정보입니다. OLED 모듈과 PN532 모듈은 I2C 버스(SDA, SCL) 라인을 공통으로 공유하여 구성합니다.

연결 모듈 모듈 핀 이름 NodeMCU 핀 (GPIO) 스펙 및 비고
SSD1306 OLED SDA / SCL D5 (GPIO14) / D6 (GPIO12) 128x64 해상도, I2C 주소 0x3C
PN532 RFID SDA / SCL D5 (GPIO14) / D6 (GPIO12) OLED와 통신선 공유
PN532 RFID RST / IRQ D1 (GPIO5) / D2 (GPIO4) IRQ 핀은 펌웨어에서 선택적 폴링 모드 동작
패시브 부저 + / - D7 (GPIO13) / GND 극성 주의(긴 다리가 +), tone() 함수 제어

※ 주의: PN532 모듈의 딥스위치(Dip Switch)는 반드시 I2C 모드에 맞게(예: 1번 ON / 2번 OFF) 기판을 확인하여 물리적으로 세팅해야 합니다.

3. 데이터베이스 스키마 설계

MariaDB 내부에 위치한 두 가지 핵심 테이블 명세입니다.

1) 사용자 관리 테이블 (`registered_cards`)

시스템에 정식으로 허가되어 등록된 카드 정보와 유저 신상 정보를 담습니다.

CREATE TABLE registered_cards (
    id INT AUTO_INCREMENT PRIMARY KEY,
    uid VARCHAR(50) UNIQUE NOT NULL,      -- RFID 카드 UID (고유키)
    username VARCHAR(50) NOT NULL,        -- 이름
    role VARCHAR(50) NOT NULL,            -- 소속 및 역할
    phone VARCHAR(20) DEFAULT '',         -- 전화번호
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

2) 출입 로그 테이블 (`access_logs`)

사용자 카드 태그 시 통과 또는 실패 정보를 보관하는 로그 테이블입니다.

CREATE TABLE access_logs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    uid VARCHAR(50) NOT NULL,
    username VARCHAR(50) DEFAULT '미등록 카드',
    role VARCHAR(50) DEFAULT '외부인',
    status VARCHAR(20) NOT NULL,          -- 'success' (승인) 또는 'failed' (거부)
    remarks VARCHAR(255) DEFAULT '',       -- 에러 및 승인 비고
    tagged_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

4. 백엔드 API 명세

모든 API는 /api.php 엔드포인트에서 action 쿼리 스트링 값을 기반으로 분기 동작합니다.

POST /api.php?action=tag
아두이노 기기에서 RFID 카드를 읽은 후 전송하는 통로입니다. DB를 대조하여 승인/거부를 리턴하고 기록을 남깁니다.
// Request Body (JSON)
{
    "uid": "04 A2 3B 8C"
}

// Response (JSON)
{
    "status": "success",          // success 또는 failed
    "username": "홍길동",
    "role": "총괄 관리자",
    "phone": "010-1234-5678",
    "remarks": "정상 출입",
    "uid": "04 A2 3B 8C"
}
GET /api.php?action=logs
대시보드 실시간 렌더링을 위해 최신 출입 로그 50개를 조회합니다.
GET /api.php?action=stats
오늘 누적된 총 태그 수, 승인 수, 실패 수치를 집계해 리턴합니다.
GET /api.php?action=users
데이터베이스에 등록된 전체 사용자 목록을 가공해 리턴합니다.
POST /api.php?action=register
웹 대시보드 모달에서 신규 사용자를 등록하거나 기존 카드의 정보를 수정(UPSERT)합니다.
// Request Body (JSON)
{
    "uid": "D3 F2 9A 4E",
    "username": "이영희",
    "role": "보안팀",
    "phone": "010-9876-5432"
}
GET /api.php?action=download_csv
전체 출입 기록을 엑셀 한글 깨짐 방지 BOM이 포함된 CSV 파일 형태로 즉시 내려받습니다.

5. 디바이스 펌웨어 설계 및 핵심 동작

디바이스의 소스코드는 c:\Users\gaeul kim\Desktop\AWS\NodeMCU_RFID_Test\NodeMCU_RFID_Test.ino 에 위치합니다. 기기가 멈추지 않고 실시간으로 시계를 돌리며 카드를 대기할 수 있도록 설계했습니다.

비블로킹(Non-Blocking) 50ms 폴링

기존 Adafruit PN532의 단순 감지 함수는 카드가 리더기에 인식될 때까지 아두이노 동작을 완전히 정지시킵니다(무한 블로킹). 이로 인해 시계 갱신 및 WiFi 연결 유지가 끊어지는 문제를 방지하기 위해 50ms 타임아웃 파라미터를 사용하는 폴링 함수를 도입했습니다.

success = nfc.readPassiveTargetID(PN532_MIFARE_ISO14443A, uid, &uidLength, 50);

카드가 없을 경우 50ms 이내에 즉시 통제권을 반환하여 루프를 돌리고 시계를 부드럽게 유지합니다. 이로 인해 물리적 IRQ(인터럽트) 핀 연결 배선 없이도 완벽한 비블로킹 동작이 가능합니다.

패시브 부저 멜로디 톤 제어

패시브 부저(D7/GPIO13)에 알맞은 주파수를 가하여 상태 알림을 연출합니다. setup() 실행 시 부저 정상 연결을 증명하는 자가진단 삐- 소리가 울립니다.

  • 승인 (APPROVED): tone(D7, 2000, 100) 2회 연속 출력 (삐빅!)
  • 거부 (DENIED): tone(D7, 800, 800) 1회 길게 출력 (삐~~~)
